Component Name

Component Type

Description

A comprehensive wrapper for analytics reports that manages layout, filtering controls, data visualization tables, and various content states (loading, error, empty).

Design System Alignment

Implements the 'Card' data display pattern, integrates 'Date Range Filter' molecule, uses system typography scale for hierarchies, and semantic colors for states.

Responsive Specifications

Mobile Adaptations

  • Report header stacks vertically (Title top, Controls bottom)
  • Data table wrapper enables horizontal scrolling
  • Pagination controls stack or simplify

Tablet Adaptations

  • Header controls may wrap but remain flex-aligned
  • Table density increases

Desktop Enhancements

  • Header aligns in a single row
  • Full width table display
  • Sticky table headers enabled

Accessibility Features

Keyboard Support

Logical tab flow through filters to table headers to pagination

Screen Reader Support

Live regions for state changes (loading/error), proper table semantics (th scope), aria-labels for actions

Focus Management

Visible focus rings on interactive elements

Aria Attributes

  • role='region'
  • aria-busy='true/false'
  • aria-live='polite'

Color Contrast

Text meets 4.5:1 ratio, borders 3:1 for inputs

Implementation Notes

Embedded Css Architecture

BEM naming convention (e.g., .report-builder__header). Flexbox for layout, CSS Grid for internal alignment.

Component Organization

Self-contained organism integrating molecular filters and atomic buttons/inputs.

Responsive Design

Container query logic simulated via media queries for component-level responsiveness.

Accessibility Compliance

WCAG 2.1 AA compliant table structure and state messaging.

Browser Support

Chrome, Edge, Firefox, Safari (Modern Evergreen)

Usage Guidelines

When To Use

For any full-page analytics report requiring data manipulation (filtering, sorting) and export.

Content Guidelines

Titles should be descriptive. Tables should max out at ~10-15 rows per page before pagination.

Placement Rules

Main content area of the dashboard layout.

Customization Options

  • Variant classes: .report-builder--sales, .report-builder--product
  • State classes: .is-loading, .is-empty, .is-error

Testing Recommendations

Browser Testing

Verify table horizontal scroll on mobile devices.

Device Testing

Check touch targets for filters and pagination on tablets.

Accessibility Testing

Test screen reader announcements when switching from Loading to Data Loaded states.

Performance Testing

Ensure layout is stable during state transitions (no cumulative layout shift).
