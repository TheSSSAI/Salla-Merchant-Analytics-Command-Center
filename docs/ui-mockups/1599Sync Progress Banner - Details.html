Component Information

Component Id

ORG-ONB-001

Component Name

Sync Progress Banner

Component Type

organism

Description

A persistent, state-aware notification component that communicates the status of long-running background data synchronization tasks. Supports real-time progress updates, time estimation, and success/failure states.

Design System Alignment

Utilizes the Semantic Color palette (Info, Success, Error), standard Spacing tokens, and Shadow depth for floating states. typography scales body/caption.

Design Tokens Used

--color-primary-600
--color-success-bg
--color-error-bg
--radius-lg
--shadow-floating
--font-family

Responsive Specifications

Mobile Adaptations

  • Floating banner becomes full width with margin
  • Description text hidden to save vertical space (Compact Mode)
  • Icon size reduced from 32px to 24px
  • Action buttons span full width for easier touch targets

Tablet Adaptations

  • Floating banner anchored to bottom-right
  • Full descriptive text visible

Desktop Enhancements

  • Hover effects on buttons
  • Optimal width of 360px for floating variant

Accessibility Features

Keyboard Support

Action buttons are keyboard accessible. Focus visible styles included.

Screen Reader Support

Uses role='status' for active sync and role='alert' for failure. Progress bar uses aria-valuenow.

Focus Management

Buttons have high contrast focus rings.

Aria Attributes

  • role='status'
  • aria-live='polite'
  • aria-label='Data synchronization progress'

Color Contrast

Text colors on success/error backgrounds vetted for 4.5:1 contrast.

Implementation Notes

Embedded Css Architecture

BEM naming convention. Variables defined in root. Dependency (Progress Bar) implemented internally.

Component Organization

Showcase includes Inline vs Floating variants and all functional states (Syncing, Calculating, Success, Error).

Responsive Design

Media queries handle the 'Compact Mode' transition automatically below 640px.

Accessibility Compliance

Native semantics and ARIA roles used. No JavaScript required for visual states.

Browser Support

Works in all modern browsers. Flexbox layout.

Usage Guidelines

When To Use

Display immediately after a user connects a store or initiates a large data import.

Content Guidelines

Keep titles action-oriented (e.g., 'Syncing orders...'). Use 'Calculating...' for estimation phase.

Placement Rules

Floating: Bottom-right for persistence. Inline: Top of Dashboard or Settings page.

Customization Options

  • Use .sync-banner--floating for global context
  • Use .sync-banner--inline for page-specific context

Testing Recommendations

Browser Testing

Verify flexbox behavior on Safari vs Chrome.

Device Testing

Check 'Compact Mode' on mobile simulation.

Accessibility Testing

Verify screen reader announces status changes (mocked via live region logic).

Performance Testing

Ensure CSS shimmer animation doesn't cause high CPU usage.
