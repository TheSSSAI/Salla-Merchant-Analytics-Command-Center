Component Information

Component Id

ORG-DATA-005

Component Name

Chart Widget

Component Type

organism

Description

A responsive data visualization container capable of rendering various chart types (Line, Bar, Area, Pie) with built-in states for loading, empty data, and interactive data points.

Design System Alignment

Utilizes semantic color palette for data series, standard card elevation/border tokens, and typography scale for headers and tooltips.

Design Tokens Used

--color-gray-500
--color-gray-900

--shadow-sm
--font-family

Responsive Specifications

Mobile Adaptations

  • Container height fixed to 250px
  • X-axis labels rotated or decimated (show every nth label)
  • Tooltips repositioned to center to prevent overflow
  • Legend moves below chart

Tablet Adaptations

  • Container uses aspect-ratio (16:9)
  • Full axis labeling enabled

Desktop Enhancements

  • Interactive hover states on data points
  • Detailed tooltips with multiple metrics
  • Action buttons in header visible without menu

Accessibility Features

Keyboard Support

Data points are focusable (tabindex=0) and show tooltips on focus

Screen Reader Support

Charts have role='img' with aria-label describing the trend. Hidden table fallback provided for complex data.

Focus Management

Focus rings on interactive data elements

Aria Attributes

  • aria-label
  • aria-hidden
  • role='graphics-document'

Color Contrast

Data colors checked against white background (3:1 for graphical objects)

CSS Implementation

See embedded style tag in HTML output

Implementation Notes

Embedded Css Architecture

BEM naming (e.g., .chart-widget__body), scoped variables for chart specific colors

Component Organization

Showcase includes Bar, Line, and Pie variants plus Loading and No Data states

Responsive Design

CSS Grid/Flexbox used for layout, SVG 'preserveAspectRatio' for scaling graphics

Accessibility Compliance

Includes visually hidden data summaries for screen readers

Browser Support

Modern browsers (CSS Grid, Flexbox, SVG)

Usage Guidelines

When To Use

To visualize trends, comparisons, or distributions of data (e.g., Sales over time, Traffic sources).

Content Guidelines

Title should include Metric + Timeframe. Y-axis should start at 0 where possible.

Placement Rules

Place in Dashboard Grids (spanning 4-12 columns) or Report Containers.

Customization Options

  • Chart Type: Bar, Line, Area, Pie
  • Height: Fixed or Aspect Ratio
  • Header Actions: Date filter, Export

Testing Recommendations

Browser Testing

Verify SVG scaling in Safari vs Chrome

Device Testing

Check tooltip touch interaction on mobile

Accessibility Testing

  • Tab through data points
  • Verify screen reader announces summary
  • Check high contrast mode visibility

Performance Testing

Monitor paint time with large datasets (SVG nodes)
